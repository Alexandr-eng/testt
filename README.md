# Проект API для мобильного приложения

Этот проект представляет собой простое API для мобильного приложения, в котором полевой сотрудник заказчика выполняет визиты в магазины.

## Установка и запуск

1. **Клонируйте репозиторий**:
   ```sh
   git clone https://github.com/Alexandr-eng/testt.git
   
Установите зависимости:


pip install -r requirements.txt
Примените миграции:

python manage.py makemigrations
python manage.py migrate
Создайте суперпользователя:

python manage.py createsuperuser

Запустите сервер разработки:

python manage.py runserver
Структура проекта
api: Приложение, содержащее модели, сериализаторы, представления и админку.

models.py: Определение моделей Worker, Store и Visit.

serializers.py: Сериализаторы для моделей.

views.py: Представления для обработки запросов.

urls.py: Маршруты для API.

admin.py: Настройка админки.

tests.py: Юнит-тесты для приложения.

API Endpoints
GET /api/stores/: Получить список торговых точек, привязанных к переданному номеру телефона.

Заголовок: Authorization: Phone {номер телефона работника}

Ответ: Список торговых точек в формате JSON.

POST /api/visit/: Выполнить посещение в торговую точку.

Заголовок: Authorization: Phone {номер телефона работника}

Тело запроса: store_id, latitude, longitude

Ответ: Информация о посещении в формате JSON.

Админка
Работник (Worker):

Создание, редактирование и удаление работников.

Поиск по имени.

Торговая точка (Store):

Создание, редактирование и удаление торговых точек.

Поиск по названию.

Посещение (Visit):

Просмотр посещений.

Поиск по имени Работника и названию Торговой точки.

Тестирование
Для тестирования API можно использовать Postman или cURL. Примеры запросов:

Получение списка торговых точек:

curl -H "Authorization: Phone 1234567890" http://127.0.0.1:8000/api/stores/


Создание посещения:

curl -X POST -H "Authorization: Phone 1234567890" -d "store_id=1&latitude=55.7558&longitude=37.6173" http://127.0.0.1:8000/api/visit/
